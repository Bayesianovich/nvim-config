# 视频 02：日常工作流

## 标题候选

- 我每天怎么用这套 Neovim 配置写代码
- 终端、Git、剪贴板都在 Neovim 里怎么串起来
- 这一套 Neovim 日常工作流，我为什么越用越顺手

## 定位

- 建议时长：8-12 分钟
- 目标观众：想看真实日常操作路径的人
- 核心目标：让观众看到“打开项目之后，我是怎么连续工作的”

## 开场钩子

> 上一期我讲的是结构，这一期我直接讲日常工作流。也就是我打开一个项目之后，终端怎么开，Git 怎么看，历史怎么翻，剪贴板怎么用，链接和路径怎么直接打开。这部分比插件列表更重要，因为它决定了你是不是愿意每天真的用它。

## 讲稿

### 第一段：我希望入口尽量少

> 我自己很在意一件事，就是常用操作的入口要少。因为一个工作流只要中间多切两次窗口，多想一次快捷键，长期下来就会变重。所以我会把最常用的动作压到非常短的路径上。

### 第二段：浮动终端

> 比如我这里最常用的就是浮动终端。在 `lua/config/keymaps.lua` 里我把 `<leader>2` 绑成了一个居中的浮动终端。这个动作的意义不是“我也能在 Neovim 开终端”，而是我想让终端变成一个随时弹出来、做完事就收回去的临时工作区。

> 你可以把它想成一个快速上下文切换器。比如跑测试、看构建输出、临时执行 Git 命令，做完就关，不需要打断当前的编辑节奏。

> 如果这个任务不是临时看一眼，而是我想把终端挂在旁边持续观察，比如看日志、看测试、边改边跑，那我就会用 `<leader>3` 打开右侧终端。这样主编辑区和终端区的职责会更清楚。

### 第三段：Git 入口

> 第二个高频入口是 Git。我这里把 `<leader>gg` 直接绑成了 `lazygit`。这和很多人的习惯一样，但我的重点不是“我也用了 lazygit”，而是我把它放到了整个工作流的核心位置。你要看状态、切分支、提交、挑 diff，都是最快路径进去。

> 如果我需要更偏查看历史的方式，我还会用 `diffview`。比如 `<leader>gH` 看当前文件历史，`<leader>gV` 打开差异视图，这些都比单纯在命令行里翻 Git 日志更直观。

### 第四段：复制粘贴历史

> 再一个我觉得很值的插件是 `yanky.nvim`，配置在 `lua/plugins/editing.lua`。我给它留了 `<leader>p` 作为 yank history 的入口。这个东西的价值不是炫技，而是当你在多个函数、多段代码之间来回摘抄时，你不再只有一个剪贴板槽位可以用。

> 这个点对长期写代码的人很关键。因为编辑效率很多时候不是快在“输入”，而是快在“组织和复用”。

### 第五段：项目内信息流

> 平时我还会用 TODO 搜索和文件历史这种信息流工具。比如项目里有 `TODO`、`FIXME`，我会直接用 `<leader>xt` 或 `<leader>xT` 把它们拉到 Trouble 里，看一眼整个项目当前有哪些待办。这个功能虽然小，但很适合在开工前快速建立上下文。

> 还有一个很日常但很省事的动作是 `gx`。如果光标下是一个链接，或者是 README、日志、报错里出现的文件路径，我可以直接按 `gx` 交给系统打开，不用自己再复制出去处理。

### 结尾

> 这一期你如果只想抄一个思路，那就是别先想着装多少插件，而是先想清楚你每天最常做的 5 件事是什么。我的答案大概就是开终端、看 Git、切上下文、复用剪贴板、扫项目待办。所以我把这些动作都压缩成了很短的路径。

## 录屏顺序

1. 从项目根目录执行 `nvim .` 进入。
2. 打开一个代码文件，先按 `<leader>2` 弹出浮动终端，再关闭。
3. 再次按 `<leader>2`，在终端里跑一个简单命令，比如测试或 `git status`。
4. 再按 `<leader>3` 打开右侧终端，说明它更适合挂日志或边改边跑。
5. 回到正常编辑界面，按 `<leader>gg` 打开 `lazygit`。
6. 在 `lazygit` 里展示未提交改动、切文件、看 diff。
7. 退出后，打开一个文件，复制几段内容，再按 `<leader>p` 展示 yank history。
8. 在项目里按 `<leader>xT`，演示 `TODO/FIX/FIXME` 列表。
9. 再按 `<leader>gH` 看当前文件历史，或者按 `<leader>gV` 打开差异视图。
10. 如果镜头够，再补一个 `gx`，直接打开光标下的链接或文件路径。

## 录屏时建议准备的素材

- 一个有未提交改动的 Git 仓库
- 至少 2-3 个源文件，方便演示复制和切换
- 若干 `TODO:` 或 `FIXME:` 注释
- 一个可以立刻执行的测试命令
- 一段可以被 `gx` 打开的 URL 或文件路径

## 这一集重点要切到的文件

- `lua/config/keymaps.lua`
- `lua/config/platform.lua`
- `lua/plugins/editing.lua`
- `lua/plugins/git-view.lua`
- `lua/plugins/todo.lua`

## 你可以直接念的总结

> 我希望 Neovim 不是一个只负责编辑文本的地方，而是我每天工作的主界面。终端、Git、复制粘贴历史、项目待办，这些都应该在同一套上下文里完成。这样一来，注意力不会被来回切碎。

## 注意事项

- 演示 `lazygit` 前先确认项目里有东西可看，不然画面会很空。
- 演示 yank history 时，最好提前准备两到三段明显不同的文本。
- 演示 `gx` 前最好准备一个本机能正常打开的链接或路径，避免录屏时卡住。
- 不要在这一集讲太多 AI，不然主题会散。
